# Benchmark data schemas

Every published number traces to a JSON file. Three shapes exist:

## `pockettune.bench.v1` — harness output (`results/*.json`)

Written by `harness/bench.py`, one file per device + sweep. Filename: `<device-slug>-<utc-stamp>.json`.

```jsonc
{
  "schema": "pockettune.bench.v1",
  "timestamp_utc": "20260712T115543Z",
  "device": {
    "serial": "…", "manufacturer": "Nothing", "model": "A142",
    "soc": "Mediatek MT6886", "android": "16", "abi": "arm64-v8a",
    "cpu_features": ["asimddp", "i8mm", "sve2", "…"],   // /proc/cpuinfo Features line
    "has_i8mm": true, "has_dotprod": true, "has_sve2": true,
    "cpu_max_khz": [2000000, "…"],                        // per-core cpufreq max
    "big_cores": [6, 7],
    "mem_total_kb": 7590548
  },
  "model": { "file": "Llama-3.2-1B-Instruct-Q4_0.gguf", "size_bytes": 773025920 },
  "params": {
    "prompt_tokens": 128, "gen_tokens": 64, "repetitions": 5,
    "threads_swept": [2, 4, 6], "cooldown_s": 120
  },
  "runs": {
    "<variant>": {                     // generic | arch | kleidiai | *-norepack | dp-*
      "rows": [ /* raw llama-bench JSON rows, incl. per-rep samples */ ],
      "summary": {                     // collapsed: test → threads → avg tok/s
        "pp128": { "2": 100.56, "4": 101.02, "6": 103.90 },
        "tg64":  { "2": 17.33,  "4": 15.20,  "6": 13.94 }
      },
      "battery_before": { "level_pct": 32, "temperature_c": 34.0 },
      "battery_after":  { "level_pct": 31, "temperature_c": 35.0 }
    }
  }
}
```

Build variants (each isolates one lever — consecutive comparison attributes the gain):

| variant | flags |
|---|---|
| `generic` | plain arm64-v8a, no feature flags |
| `arch` | `-march=armv8.2-a+dotprod+i8mm` |
| `kleidiai` | arch flags + `-DGGML_CPU_KLEIDIAI=ON` |
| `arch-norepack` / `kleidiai-norepack` | as above with llama.cpp's Q4_0 repack disabled |
| `dp-arch` / `dp-kleidiai` | dotprod-only (no i8mm) — the Pixel 7a target |

## `pockettune.evidence.v1` — app bundle (`app/src/data/evidence.json`)

Generated by `uv run tools/make_app_evidence.py`: the same runs with raw rows dropped,
keeping `summary`, device identity, params, and the `source` filename so every in-app
chart cites its `results/` file. Regenerate whenever `results/` changes.

## In-app sweep history (on-device only)

The app persists tune runs to its private storage (`pockettune-state.json`) as
`TuneRun` objects — see `app/src/types.ts`. Same measurement idea (fixed pp/tg,
repetitions via llama.rn `bench()`), plus `tokensPerJoule` sampled from
`/sys/class/power_supply/battery` when readable.
